🔎 Imperative API — Tool Registrations

Tool workflow pattern
This page registers three tools that support iterative trip configuration: configure_package updates both the UI and state with each call, check_prerequisites verifies booking readiness, and get_price_estimate provides read-only "what if" pricing. The agent can call configure_package multiple times as the traveler refines their options — each call cleanly replaces the previous configuration.
configure_package — primary configuration tool
Accepts era, package template, duration, and add-ons (all optional with sensible defaults). Resolves by ID or name, calculates pricing with era multiplier and duration adjustment, updates the visible UI, and saves to ChronoShared.updateState("packageConfig", ...). Designed for iterative use — each call fully replaces the prior configuration.
check_prerequisites — readiness verification
Checks each prerequisite's status against ChronoShared.getClearanceStatus() and the traveler's clearance level vs. the era's requirement. Returns structured guidance directing the agent to suggest the Clearance page or Profile page as needed.
get_price_estimate — read-only pricing
Same calculation logic as configure_package but does not update the UI or save to state. Use for hypothetical comparisons like "what's the price difference between 7 and 14 days?"
